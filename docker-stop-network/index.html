docker 解决 stop kill 假死和docker-compose的 network has active endpoints 问题

docker docker-compose
docker docker-compose
发布日期:   2020-11-15
更新日期:   2020-11-21
文章字数:   3.8k
阅读时长:   22 分

问题

使用docker-compose down的时候发现network不能停止

[root@176 website-cdn]# docker-compose down
Stopping hw_nginx  ... done
Stopping hw_django ... done
Stopping hw_mysql  ... done
Removing hw_django ... done
Removing hw_mysql  ... done
Removing network website-cdn_django_network
ERROR: error while removing network: network website-cdn_django_network id 0f44a37851fb3e4160676d1e3620fd56ace3bc098686851ce5ca772c128f0559 has active endpoints
[root@176 website-cdn]# docker-compose ps

解决

  1. docker network inspect {network}获得{endpoint-name}，即docker-compse.yml中的container_name
  2. docker network disconnect -f {network} {endpoint-name}
  3. 重启docker-compose up -d
  4. 关闭docker-compose down
  5. 估计有些人的问题已经解决，但是我发现进程依然存在，已经造成假死
  6. docker进行stop，失败
  7. 容器挂载对应目录/var/lib/docker/containers进行 root 删除，提示设备或资源忙，失败
  8. 取消各种挂载umount /data/sys/var/docker/containers/，失败
  9. 相关的进程号lsof |grep "挂载目录"直接kill -9进程号失败
  10. 最终，重启 docker 解决sudo service docker restart， _ ._!!!
